The calendar is the heart of Roadcase. It gives you a unified view of everything happening across your artists — shows, rehearsals, meetings, interviews, writing sessions, and more.

## Calendar views

Roadcase offers three calendar views:

| View | Best for |
|---|---|
| **Month** | Getting an overview of your schedule at a glance |
| **Week** | Planning a specific week in detail |
| **Day** | Focusing on a single day's events |

### Filtering

When you belong to multiple artists, the calendar displays events from all of them by default. Use the artist filter to show or hide events for specific artists. Each artist's events are color-coded by their avatar color, making it easy to distinguish between acts.

## Event types

Roadcase supports a variety of event types to cover every aspect of your schedule:

### Show

The most common event type. A show represents a performance at a venue. Shows are the richest event type in Roadcase, with support for schedules, deals, settlements, guest lists, advances, and more.

[Learn more about shows](/tour-planning/shows)

### Rehearsal

Track band rehearsals with date, time, and location.

### Meeting

Business meetings with managers, labels, agents, or other industry contacts.

### Interview

Press and media interviews. Useful for tracking promo schedules around tours.

### Writing session

Songwriting sessions with co-writer tracking. Keep a record of when and with whom you wrote.

### Travel

Track a travel leg between two locations. Each travel event includes departure and arrival locations with details like distance and estimated drive time. Useful for planning how your team gets between shows.

### Lodging

Track overnight accommodations. Lodging events support several types:

| Type | Description |
|---|---|
| **Hotel** | Standard hotel booking |
| **Airbnb** | Short-term rental |
| **Couch surf** | Staying with friends or contacts |
| **RV park** | For artists touring in RVs or buses |
| **Other** | Any other arrangement |

### Hold

A tentative date hold. Use holds to mark dates that are being discussed but not yet confirmed. This is common during the booking process when a promoter or venue is considering a date.

### Block date

Personal unavailability. Each team member can block dates to mark when they are not available. Administrators can see all block dates across the team and are alerted to conflicts when a block date overlaps with an event.

[Learn more about availability](/guides/availability-and-conflicts)

### Custom event types

Create your own event types with custom names, colors, and icons. Useful for things like photo shoots, video shoots, listening parties, or anything else specific to your operation.

[Learn more about custom event types](/core-concepts/custom-event-types)

## Creating events

To create an event:

1. Click **New event** on the calendar (or click directly on a date)
2. Select the event type
3. Fill in the details — at minimum, a title and date
4. Save

Events can span multiple days. For recurring events, create each occurrence individually.

## Event details

Every event has these common fields:

| Field | Description |
|---|---|
| **Title** | The name of the event |
| **Date & time** | When it starts and ends |
| **All day** | Toggle for all-day events without specific times |
| **Notes** | Rich-text notes visible to the team |

Shows have many additional fields. See [Shows](/tour-planning/shows) for details.

## Editing and deleting events

Administrators can edit or delete any event. Click on an event in the calendar to open it, then use the edit or delete options.

When an event is updated, team members who have notifications enabled receive a notification about the change. See [Notifications](/account/notifications) for notification preferences.
